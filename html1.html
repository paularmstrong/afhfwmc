DOM

That's just the HTML Elements, right?

No.

DOM: Set of Objects and Interfaces for Accessing and Manipulating Document Objects

The DOM Structure Model

  • Document
  • DocumentFragment
  • DocumentType
  • EntityReference
  • Element
  • Attr
  • ProcessingInstruction
  • Comment
  • Text
  • CDataSection
  • Entity
  • Notation

Document

  • Single point-of-entry that represents the entire HTML or XML document.
  • Root of the document tree.
  1. http://www.w3.org/TR/2004/REC-DOM-Level-3-Core-20040407/core.html#i-Document

DocumentFragment

  • Lightweight Document object.
  • Useful for extracting portions of a document.
  • Creating new parts of a document.
  1. http://www.w3.org/TR/2004/REC-DOM-Level-3-Core-20040407/core.html#ID-B63ED1A3

DocumentType

  • The doctype of the Document.
  • readonly
<!DOCTYPE HTML>
  1. http://www.w3.org/TR/2004/REC-DOM-Level-3-Core-20040407/core.html#ID-412266927

EntityReference

  • Reference to an Entity in the tree.
  1. http://www.w3.org/TR/2004/REC-DOM-Level-3-Core-20040407/core.html#ID-11C98490

Element

  • An HTML or XML Element.
<div></div>
  1. http://www.w3.org/TR/2004/REC-DOM-Level-3-Core-20040407/core.html#ID-745549614

Attr

  • An attribute on an (HTML or XML) Element.
  • Not considered part of the document tree.
<div class="foobar"></div>
  1. http://www.w3.org/TR/2004/REC-DOM-Level-3-Core-20040407/core.html#ID-637646024

ProcessingInstruction

  • A way to keep processor-specific information in the document.
  • Not actual document content.
<?php echo "foobar"; ?>
  1. http://www.w3.org/TR/2004/REC-DOM-Level-3-Core-20040407/core.html#ID-1004215813

Comment

  • A piece of CharacterData.
  • Should be ignored when consuming.
<!-- hi mom! -->
  1. http://www.w3.org/TR/2004/REC-DOM-Level-3-Core-20040407/core.html#ID-1728279322

Text

  • A piece of CharacterData.
  • Textual content of an Element or Attr.
  • Only 1 text node per text block.
<p>hi mom!</p>
  1. http://www.w3.org/TR/2004/REC-DOM-Level-3-Core-20040407/core.html#ID-1312295772

CDATASection

  • Used to escape blocks of text that could be considered markup.
  • Useful for including XML fragments without needing to escape characters.
<![CDATA[
    <foo> will not create an element of type 'foo'
]]>

Behavior

  • Replace character entities with characters,
  • Ignore line feeds,
  • Replace each carriage return or tab with a single space.
  1. http://www.w3.org/TR/2004/REC-DOM-Level-3-Core-20040407/core.html#ID-667469212

Entity

  • Usually an HTML entity, parsed or unparsed.
&amp;
&lt;
&gt;
…
  1. http://www.w3.org/TR/2004/REC-DOM-Level-3-Core-20040407/core.html#ID-527DCFF2

Notation

  • The format of an unparsed entity.
  • readonly
  • You'll only ever need this when parsing XML or custom DTDs.
  1. http://www.w3.org/TR/2004/REC-DOM-Level-3-Core-20040407/core.html#ID-5431D1B9
